# Tailwind Nextjs 博客模板

这是一个基于 [Next.js](https://nextjs.org/) 和 [Tailwind CSS](https://tailwindcss.com/) 的博客启动模板。这可能是功能最丰富的 Next.js Markdown 博客模板。开箱即配，与最新的技术完美结合，让技术写作变得轻松。易于配置和定制。是替代现有 Jekyll 和 Hugo 个人博客的绝佳选择。

查看下面的文档开始使用吧。

遇到问题？可以查看 [常见问题页面](https://github.com/timlrx/tailwind-nextjs-starter-blog/wiki) 并搜索以往的问题。如果之前没有发布过，欢迎提交新问题。

有新功能建议？可以查看过去的讨论，看看是否已经有人提过。否则欢迎开始新的讨论。所有想法都欢迎！

## 初衷

我想把现有的博客移植到 Nextjs 和 Tailwind CSS，但没有现成的、易于使用的模板，所以决定自己创建一个。设计借鉴了 [Tailwindlabs 博客](https://github.com/tailwindlabs/blog.tailwindcss.com)。

我希望它像 [beautiful-jekyll](https://github.com/daattali/beautiful-jekyll) 和 [Hugo Academic](https://github.com/wowchemy/wowchemy-hugo-modules) 这些流行的博客模板一样功能丰富，但同时结合 React 生态系统和当前 Web 开发最佳实践的优点。

## 功能特点

- 使用 [Tailwind 3.0](https://tailwindcss.com/blog/tailwindcss-v3) 和主色属性，轻松定制样式
- 近乎完美的 Lighthouse 评分 - [Lighthouse 报告](https://www.webpagetest.org/result/210111_DiC1_08f3670c3430bf4a9b76fc3b927716c5/)
- 轻量级，首屏 JS 仅 45kB，生产构建使用 Preact
- 移动端友好
- 浅色和深色主题
- 使用 [Fontsource](https://fontsource.org/) 自托管字体
- 支持 [plausible](https://plausible.io/)、[simple analytics](https://simpleanalytics.com/) 和 Google Analytics
- [MDX - 在 Markdown 文档中写 JSX！](https://mdxjs.com/)
- 通过 [rehype-prism-plus](https://github.com/timlrx/rehype-prism-plus) 实现服务端语法高亮，支持行号和行高亮
- 通过 [KaTeX](https://katex.org/) 支持数学公式
- 通过 [rehype-citation](https://github.com/timlrx/rehype-citation) 支持引用和参考文献
- 通过 [next/image](https://nextjs.org/docs/basic-features/image-optimization) 自动优化图片
- 通过 [mdx-bundler](https://github.com/kentcdodds/mdx-bundler) 灵活获取数据
- 支持标签 - 每个唯一标签都有自己的页面
- 支持多作者
- 博客模板
- 目录组件
- 支持博客文章的嵌套路由
- 通讯组件，支持 Mailchimp、Buttondown、Convertkit、Klaviyo、Revue、Emailoctopus 和 Beehiiv
- 支持 [giscus](https://github.com/laymonage/giscus)、[utterances](https://github.com/utterance/utterances) 或 Disqus
- 项目页面
- 预配置的安全头
- SEO 友好，支持 RSS 订阅、站点地图等！

## 快速开始

1. 运行 `npx degit https://github.com/timlrx/tailwind-nextjs-starter-blog.git` 克隆 JS 版本（或 TS 版本 `npx degit timlrx/tailwind-nextjs-starter-blog#typescript`）
2. 修改 `siteMetadata.js`（网站相关信息）
3. 如果你想使用其他分析工具或 Giscus 以外的评论方案，修改 `next.config.js` 中的内容安全策略
4. 修改 `authors/default.md`（主要作者信息）
5. 修改 `projectsData.ts`
6. 修改 `headerNavLinks.ts` 来自定义导航链接
7. 添加博客文章
8. 部署到 Vercel

## 开发

首先，运行开发服务器：

```bash
npm start
# 或
npm run dev
```

在浏览器中打开 [http://localhost:3000](http://localhost:3000) 查看结果。

你可以编辑页面来开始自定义。通过修改 `pages/index.js`，页面会自动更新。

## 扩展 / 自定义

`data/siteMetadata.js` - 包含大多数网站相关信息，需要根据你的需求修改。

`data/authors/default.md` - 默认作者信息（必填）。其他作者可以添加到 `data/authors` 目录。

`data/projectsData.js` - 用于在项目页面生成样式卡片的数据。

`data/headerNavLinks.js` - 导航链接。

`data/logo.svg` - 替换成你自己的 Logo。

`data/blog` - 替换成你自己的博客文章。

`public/static` - 存放图片和图标等静态资源。

`tailwind.config.js` 和 `css/tailwind.css` - 包含 Tailwind 样式表，可以修改以改变网站整体外观。

`css/prism.css` - 控制与代码块相关的样式。可以自定义它并使用你喜欢的 prismjs 主题，如 [prism 主题](https://github.com/PrismJS/prism-themes)。

`components/social-icons` - 要添加其他图标，只需从 [Simple Icons](https://simpleicons.org/) 复制 SVG 文件并在 `index.js` 中映射。其他图标使用 [heroicons](https://heroicons.com/)。

`components/MDXComponents.js` - 在这里指定你自己的 JSX 代码或 React 组件。然后你可以直接在 `.mdx` 或 `.md` 文件中调用它们。默认提供自定义链接和图片组件。

`layouts` - 页面使用的主要模板。

`pages` - 路由页面。阅读 [Next.js 文档](https://nextjs.org/docs) 了解更多。

`next.config.js` - Next.js 相关配置。如果你想从其他域名加载脚本、图片等，需要修改内容安全策略。

## 文章

### Frontmatter

Frontmatter 遵循 [Hugo 的标准](https://gohugo.io/content-management/front-matter/)。

目前支持 10 个字段。

```
title (必填)
date (必填)
tags (必填，可以是空数组)
lastmod (可选)
draft (可选)
summary (可选)
images (可选，如果未提供，默认为 siteMetadata 配置中的 socialBanner)
authors (可选列表，应对应 `data/authors` 中的文件名。如果未指定则使用 'default')
layout (可选列表，应对应 `data/layouts` 中的文件名)
canonicalUrl (可选，用于 SEO 的规范 URL)
```

这是文章 frontmatter 的示例：

```
---
title: 'Tailwind Nextjs 博客模板介绍'
date: '2021-01-12'
lastmod: '2021-01-18'
tags: ['next-js', 'tailwind', '教程']
draft: false
summary: '想要一个开箱即用、性能出色的模板？快来试试！'
images: ['/static/images/canada/mountains.jpg', '/static/images/canada/toronto.jpg']
authors: ['default']
layout: PostLayout
canonicalUrl: https://example.com/blog/introducing-tailwind-nextjs-starter-blog
---
```

### 写作

运行 `node ./scripts/compose.js` 来创建新文章。

按照交互式提示生成带有预填充 front matter 的文章。

## 部署

**Vercel**

部署模板最简单的方法是使用 Next.js 创建者提供的 [Vercel 平台](https://vercel.com)。查看 [Next.js 部署文档](https://nextjs.org/docs/deployment) 了解更多详情。

**Netlify / GitHub Pages / Firebase 等**

由于模板使用 `next/image` 进行图片优化，在 [Netlify](https://www.netlify.com/) 或 [GitHub Pages](https://pages.github.com/) 等其他流行的静态托管网站部署时，需要进行额外配置。需要使用 Imgix、Cloudinary 或 Akamai 等替代图片优化提供商。或者，将 `next/image` 组件替换为标准 `<img>` 标签。详见 [`next/image` 文档](https://nextjs.org/docs/basic-features/image-optimization)。

通讯组件使用的 API 路由无法在静态站点导出中使用。你需要使用表单 API 端点提供商，并相应地替换通讯组件中的路由。Netlify 等其他托管平台也提供替代解决方案，请参阅其文档了解更多信息。
