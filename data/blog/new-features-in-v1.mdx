## 概述

这是一篇关于 v1.0 新功能的文章。新功能包括：

<TOCInline toc={props.toc} exclude="概述" toHeading={2} />

尽管添加了所有新功能，首屏 JS 从 43kB 减少到了 39kB！[^1]

[^1]: 随着 Next.js 12 的更新，首屏 JS 增加到了 45kB。

如果你从 v0 版本迁移过来，请参阅下面的[升级指南](#升级指南)。

## 主题颜色

你可以通过修改 tailwind 配置文件的 primary 属性来轻松修改主题颜色：

```js:tailwind.config.js
theme: {
    colors: {
      primary: colors.teal,
      gray: colors.neutral,
      ...
    }
  ...
}
```

primary 颜色属性应该被赋值一个包含 50、100、200...900 键及其对应颜色代码值的对象。

Tailwind 内置了出色的默认调色板，可用于为主题化你自己的网站。查看[自定义颜色文档页面](https://tailwindcss.com/docs/customizing-colors)了解完整的选项范围。

从 v1 迁移？你可以通过将 `primary` 设置为 `colors.sky`（Tailwind 2.2.2 及以上版本，否则为 `colors.lightBlue`）并将 gray 改为 `colors.gray` 来恢复到之前的主题。

从 v1.1.2+ 开始，你还可以通过修改 `css/prism.css` 样式表来轻松自定义代码块的样式。Token 类名与 prismjs 兼容，因此你可以复制并改编 prismjs 样式表中的 token 样式，如 [prism 主题](https://github.com/PrismJS/prism-themes)。

## Xdm MDX 编译器

我们将 MDX bundler 从 [next-mdx-remote](https://github.com/hashicorp/next-mdx-remote) 切换到了 [mdx-bundler](https://github.com/kentcdodds/mdx-bundler)。
这在底层使用 [xdm](https://github.com/wooorm/xdm)，即最新的 micromark 3 和 remark、rehype 库。

**警告：** 如果你正在使用自定义的 remark 或 rehype 库，请升级到与 micromark 3 兼容的版本。如果你正在升级，请删除 `node_modules` 和 `package-lock.json` 以避免过去依赖相关的问题。

[xdm](https://github.com/wooorm/xdm) 相比 [@mdx-js/mdx](https://github.com/mdx-js/mdx)（next-mdx-remote 内部使用的编译器）有许多改进，但可能会有一些破坏性行为变化。
请检查你的 markdown 输出以验证。

一些新的可能性包括直接在 mdx 文件中使用 import 语法加载组件，并包含可以在构建步骤中编译和打包的 js 代码。

例如，以下 jsx 代码片段可以直接在 MDX 文件中使用来渲染页面标题组件：

```jsx
// 如果使用 js，可以 import PageTitle from './components/PageTitle.js'
import PageTitle from './components/PageTitle.tsx'
;<PageTitle> 在 MDX 中使用 JSX 组件 </PageTitle>
```

默认配置相对于 `components` 目录解析所有组件。

**注意**：
需要外部图片加载器的组件也需要额外的 esbuild 配置。
依赖于 Next.js `Link` 组件等全局应用状态生命周期的组件在这种设置下也无法工作，因为每个 mdx 文件都是独立构建的。
对于这种情况，最好使用组件替换。

## 目录组件

灵感来自 [Docusaurus](https://docusaurus.io/docs/next/markdown-features/inline-toc) 和 Gatsby 的 [gatsby-remark-table-of-contents](https://www.gatsbyjs.com/plugins/gatsby-remark-table-of-contents/)，
包含文档所有顶级标题的 `toc` 变量会传递给 MDX 文件，可以相应地进行样式设置。
为了简化目录（TOC）生成，你可以使用现有的 `TOCInline` 组件。

例如，这篇文章的目录是用以下代码生成的：

```jsx
<TOCInline toc={props.toc} exclude="概述" toHeading={2} />
```

你可以通过配置 `fromHeading` 和 `toHeading` props 来自定义显示的标题，或者通过向 `exclude` prop 传递字符串或字符串数组来排除特定标题。默认情况下，所有深度为 3 或更小的标题都会缩进。可以通过更改 `indentDepth` 属性来配置。
`asDisclosure` prop 可用于在可展开的 disclosure 元素中渲染目录。

以下是目录在 disclosure 元素中的完整渲染：

```jsx
<TOCInline toc={props.toc} asDisclosure />
```

## 布局

你可以通过配置 frontmatter 字段将 mdx 博客内容映射到布局组件。例如，这篇文章就是用新的 `PostSimple` 布局编写的！

### 添加新模板

布局模板存储在 `./layouts` 文件夹中。你可以在此文件夹中添加要映射到 markdown 内容的 React 组件。
组件文件名必须与 markdown frontmatter `layout` 字段中指定的一致。

唯一必需的字段是包含渲染后的 MDX 内容的 `children`，不过你可能希望传入 frontMatter 内容并在模板中渲染它。

你可以配置模板以接收其他字段 - 请参阅 `PostLayout` 组件作为示例。

## 代码块复制按钮

将鼠标悬停在代码块上，你会看到一个 GitHub 风格的复制按钮！你可以修改 `./components/Pre.js` 进一步自定义它。
该组件被传递给 `MDXComponents` 并修改所有 `<pre>` 块。

## 行高亮和行号

感谢新的 [rehype-prism-plus 插件](https://github.com/timlrx/rehype-prism-plus)，现在支持行高亮和行号。

以下 JavaScript 代码块：

````
```js {1, 3-4} showLineNumbers
var num1, num2, sum
num1 = prompt('请输入第一个数字')
num2 = prompt('请输入第二个数字')
sum = parseInt(num1) + parseInt(num2) // "+" 表示相加
alert('总和 = ' + sum) // "+" 表示拼接成字符串
```
````

显示效果：

## 多作者

作者信息现在从 `siteMetadata.js` 中分离出来，作为 markdown 文件存储在独立的 `data/authors` 文件夹中。至少，你需要有一个包含作者信息的 `default.md` 文件。你可以根据需要创建其他文件，文件名将用作对作者的引用。

## 通讯组件 (v1.1.3)

v1.1.3 中引入的通讯组件为你提供了一种轻松建立受众群体的简单方法。它与以下提供商集成：

- [Mailchimp](https://mailchimp.com/)
- [Buttondown](https://buttondown.email/)
- [Convertkit](https://convertkit.com/)

要使用它，请在配置文件中指定你要使用的提供商，并将必要的环境变量添加到 `.env` 文件中。
